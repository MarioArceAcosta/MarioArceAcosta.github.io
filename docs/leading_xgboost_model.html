1 Abstract

This study estimates the extent of education–occupation mismatch and the associated wage penalties for undocumented college graduates. Using data from the American Community Survey (ACS), we classify workers as vertically mismatched (higher educational attainment than is typical for the occupation) or horizontally mismatched (field of degree is not typical for the occupation). Because the ACS does not identify undocumented status, we train machine learning models on the Survey of Income and Program Participation (SIPP) and use the predictions to impute status in the ACS. This approach enables new analyses of labor market outcomes for undocumented college graduates in nationally representative surveys. Results indicate that undocumented status is linked to higher rates of both vertical and horizontal mismatch, as well as wage penalties of roughly 4–7 percent. These penalties are smaller among the DACA-eligible and in states with more inclusive immigrant policy climates.

2 Purpose

This file is the leading XGBoost prediction workflow. It trains the SIPP donor model, scores ACS, validates the target-like threshold choice, and exports the Stata-facing prediction file. Current official threshold rule:

  • Train the XGBoost model on the SIPP donor sample.

  • Define quartile thresholds using the target-like SIPP test sample only.

  • Export those target-like threshold groups to ACS using the downstream names:

    • gbm_undocu_p

    • gbm_undocu_q

    • gbm_high_prob

    • gbm_low_prob

    • gbm_high_recall

9 Create XGBoost Matrices

XGBoost requires numeric matrices. The variables in features are already coded numerically, but mutate(across(…, as.numeric)) makes that explicit.

  select(all_of(features)) %>%
  mutate(across(everything(), as.numeric)) %>%
  as.matrix()

  select(all_of(features)) %>%
  mutate(across(everything(), as.numeric)) %>%
  as.matrix()

acs_matrix <- acs %>%
  select(all_of(features)) %>%
  mutate(across(everything(), as.numeric)) %>%
  as.matrix()

10 Tune XGBoost Model Using Cross-Validated AUC

For each grid row, xgb.cv chooses the best number of boosting rounds usingheld-out folds from the training sample. We then choose the grid row with the highest cross-validated test AUC.

Code
xgb_grid$optimal_rounds <- NA_integer_
xgb_grid$cv_auc <- NA_real_

set.seed(seed)

for (i in 1:nrow(xgb_grid)) {
  params_i <- list(
    objective = "binary:logistic",
    eval_metric = "auc",
    eta = xgb_grid$eta[i],
    max_depth = xgb_grid$max_depth[i],
    subsample = xgb_grid$subsample[i],
    colsample_bytree = xgb_grid$colsample_bytree[i]
  )

  cv_i <- xgb.cv(
    params = params_i,
    data = dtrain,
    nrounds = xgb_nrounds,
    nfold = xgb_nfold,
    early_stopping_rounds = xgb_early_stopping_rounds,
    maximize = TRUE,
    verbose = 0
  )

  eval_log_i <- cv_i$evaluation_log

  xgb_grid$cv_auc[i] <- max(eval_log_i$test_auc_mean)
  xgb_grid$optimal_rounds[i] <- eval_log_i$iter[which.max(eval_log_i$test_auc_mean)]
}

xgb_grid <- xgb_grid %>%
  arrange(desc(cv_auc))

14 Define Comparison Quartiles Using Target-like SIPP Test Sample

Threshold reference sample: SIPP test observations that match the ACS target population: college graduates, employed, ages 22-55.

Code
sipp_targetlike_test <- test_xgb %>%
  filter(
    college == 1,
    employed == 1,
    age >= 22,
    age <= 55
  )

xgb_targetlike_breaks <- quantile(
  sipp_targetlike_test$gbm_undocu_p,
  probs = c(0, .25, .50, .75, 1),
  na.rm = TRUE
)

xgb_targetlike_breaks[1] <- -Inf
xgb_targetlike_breaks[5] <- Inf

sipp_targetlike_test$gbm_undocu_q <- cut(
  sipp_targetlike_test$gbm_undocu_p,
  breaks = xgb_targetlike_breaks,
  include.lowest = TRUE,
  labels = c("Q1", "Q2", "Q3", "Q4")
)

acs$gbm_undocu_q <- cut(

  include.lowest = TRUE,

15 Define High-Recall Threshold Using Target-like SIPP Test Sample

This threshold mirrors the cleaned GBM file: it is the lowest predicted probability among the top 75% of true positives in the target-like SIPP test sample.

Code
top_75_targetlike <- sipp_targetlike_test %>%
  filter(undocu_likely == 1) %>%
  arrange(desc(gbm_undocu_p)) %>%
  mutate(
    rank = row_number(),
    cumulative_share_of_positives = rank / n(),
    top_75 = cumulative_share_of_positives <= 0.75
  ) %>%
  filter(top_75 == TRUE)

threshold_75_xgb <- min(top_75_targetlike$gbm_undocu_p)

tibble(
  threshold = "Target-like high recall",
  probability_cutoff = threshold_75_xgb
) %>%
  kable(digits = 4)

18 Export ACS Predictions for Stata

Export only the downstream GBM-named variables expected by the Stata pipeline. Even though the model is XGBoost, the variable names stay gbm_* so that the downstream Stata regression files do not need to change.

Code
acs_gbm_export <- acs %>%
  mutate(
    gbm_high_prob = ifelse(gbm_undocu_q == "Q4", 1, 0),

  select(
    gbm_undocu_p,

# Abstract

This study estimates the extent of education–occupation mismatch and the associated wage penalties for undocumented college graduates. Using data from the American Community Survey (ACS), we classify workers as vertically mismatched (higher educational attainment than is typical for the occupation) or horizontally mismatched (field of degree is not typical for the occupation). Because the ACS does not identify undocumented status, we train machine learning models on the Survey of Income and Program Participation (SIPP) and use the predictions to impute status in the ACS. This approach enables new analyses of labor market outcomes for undocumented college graduates in nationally representative surveys. Results indicate that undocumented status is linked to higher rates of both vertical and horizontal mismatch, as well as wage penalties of roughly 4–7 percent. These penalties are smaller among the DACA-eligible and in states with more inclusive immigrant policy climates.

# Purpose

This file is the leading XGBoost prediction workflow. It trains the SIPP donor model, scores ACS, validates the target-like threshold choice, and exports the Stata-facing prediction file. Current official threshold rule:

-   Train the XGBoost model on the SIPP donor sample.

-   Define quartile thresholds using the target-like SIPP test sample only.

-   Export those target-like threshold groups to ACS using the downstream names:

    -   gbm_undocu_p

    -   gbm_undocu_q

    -   gbm_high_prob

    -   gbm_low_prob

    -   gbm_high_recall

# Create XGBoost Matrices

XGBoost requires numeric matrices. The variables in features are already coded numerically, but mutate(across(..., as.numeric)) makes that explicit.

```{r}
train_matrix <- train_xgb %>%
  select(all_of(features)) %>%
  mutate(across(everything(), as.numeric)) %>%
  as.matrix()

test_matrix <- test_xgb %>%
  select(all_of(features)) %>%
  mutate(across(everything(), as.numeric)) %>%
  as.matrix()

acs_matrix <- acs %>%
  select(all_of(features)) %>%
  mutate(across(everything(), as.numeric)) %>%
  as.matrix()

train_y <- as.numeric(train_xgb$undocu_likely)
test_y <- as.numeric(test_xgb$undocu_likely)

dtrain <- xgb.DMatrix(data = train_matrix, label = train_y)
dtest  <- xgb.DMatrix(data = test_matrix, label = test_y)
dacs   <- xgb.DMatrix(data = acs_matrix)
```

# Tune XGBoost Model Using Cross-Validated AUC

For each grid row, xgb.cv chooses the best number of boosting rounds usingheld-out folds from the training sample. We then choose the grid row with the highest cross-validated test AUC.

```{r}
#| cache: true
xgb_grid$optimal_rounds <- NA_integer_
xgb_grid$cv_auc <- NA_real_

set.seed(seed)

for (i in 1:nrow(xgb_grid)) {
  params_i <- list(
    objective = "binary:logistic",
    eval_metric = "auc",
    eta = xgb_grid$eta[i],
    max_depth = xgb_grid$max_depth[i],
    subsample = xgb_grid$subsample[i],
    colsample_bytree = xgb_grid$colsample_bytree[i]
  )

  cv_i <- xgb.cv(
    params = params_i,
    data = dtrain,
    nrounds = xgb_nrounds,
    nfold = xgb_nfold,
    early_stopping_rounds = xgb_early_stopping_rounds,
    maximize = TRUE,
    verbose = 0
  )

  eval_log_i <- cv_i$evaluation_log

  xgb_grid$cv_auc[i] <- max(eval_log_i$test_auc_mean)
  xgb_grid$optimal_rounds[i] <- eval_log_i$iter[which.max(eval_log_i$test_auc_mean)]
}

xgb_grid <- xgb_grid %>%
  arrange(desc(cv_auc))

kable(xgb_grid, digits = 4)
```

# Train Final XGBoost Model Using Best Grid Row

```{r}
#| cache: true
best_xgb <- xgb_grid[1, ]

final_params <- list(
  objective = "binary:logistic",
  eval_metric = "auc",
  eta = best_xgb$eta,
  max_depth = best_xgb$max_depth,
  subsample = best_xgb$subsample,
  colsample_bytree = best_xgb$colsample_bytree
)

set.seed(seed)

xgb_model <- xgb.train(
  params = final_params,
  data = dtrain,
  nrounds = best_xgb$optimal_rounds,
```

# Define Comparison Quartiles Using Target-like SIPP Test Sample

Threshold reference sample: SIPP test observations that match the ACS target population: college graduates, employed, ages 22-55.

```{r}
sipp_targetlike_test <- test_xgb %>%
  filter(
    college == 1,
    employed == 1,
    age >= 22,
    age <= 55
  )

xgb_targetlike_breaks <- quantile(
  sipp_targetlike_test$gbm_undocu_p,
  probs = c(0, .25, .50, .75, 1),
  na.rm = TRUE
)

xgb_targetlike_breaks[1] <- -Inf
xgb_targetlike_breaks[5] <- Inf

sipp_targetlike_test$gbm_undocu_q <- cut(
  sipp_targetlike_test$gbm_undocu_p,
  breaks = xgb_targetlike_breaks,
  include.lowest = TRUE,
  labels = c("Q1", "Q2", "Q3", "Q4")
)

acs$gbm_undocu_q <- cut(
  acs$gbm_undocu_p,
  breaks = xgb_targetlike_breaks,
  include.lowest = TRUE,
  labels = c("Q1", "Q2", "Q3", "Q4")
)

tibble(
  cutoff = c("Q1/Q2", "Q2/Q3", "Q3/Q4"),
  probability_cutoff = c(
    xgb_targetlike_breaks[2],
    xgb_targetlike_breaks[3],
    xgb_targetlike_breaks[4]
  )
) %>%
  kable(digits = 4)

xgb_targetlike_sipp_quartiles <- sipp_targetlike_test %>%
  group_by(gbm_undocu_q) %>%
  summarise(
    n = n(),
    positives = sum(undocu_likely == 1),
    precision = mean(undocu_likely == 1),
    mean_probability = mean(gbm_undocu_p),
    .groups = "drop"
  )

xgb_acs_quartiles <- acs %>%
  group_by(gbm_undocu_q) %>%
  summarise(
    n = n(),
    share = n() / nrow(acs),
    mean_probability = mean(gbm_undocu_p),
    .groups = "drop"
  )

kable(xgb_targetlike_sipp_quartiles, digits = 4)
kable(xgb_acs_quartiles, digits = 4)
```

# Define High-Recall Threshold Using Target-like SIPP Test Sample

This threshold mirrors the cleaned GBM file: it is the lowest predicted probability among the top 75% of true positives in the target-like SIPP test sample.

```{r}
top_75_targetlike <- sipp_targetlike_test %>%
  filter(undocu_likely == 1) %>%
  arrange(desc(gbm_undocu_p)) %>%
  mutate(
    rank = row_number(),
    cumulative_share_of_positives = rank / n(),
    top_75 = cumulative_share_of_positives <= 0.75
  ) %>%
  filter(top_75 == TRUE)

threshold_75_xgb <- min(top_75_targetlike$gbm_undocu_p)

tibble(
  threshold = "Target-like high recall",
  probability_cutoff = threshold_75_xgb
) %>%
  kable(digits = 4)
```

# Export ACS Predictions for Stata

Export only the downstream GBM-named variables expected by the Stata pipeline. Even though the model is XGBoost, the variable names stay gbm\_\* so that the downstream Stata regression files do not need to change.

```{r}
acs_gbm_export <- acs %>%
  mutate(
    gbm_high_prob = ifelse(gbm_undocu_q == "Q4", 1, 0),
    gbm_low_prob = ifelse(gbm_undocu_q == "Q1", 1, 0),
    gbm_high_recall = ifelse(gbm_undocu_p >= threshold_75_xgb, 1, 0)
  ) %>%
  select(
    gbm_undocu_p,
    gbm_undocu_q,
    gbm_high_prob,
    gbm_low_prob,
    gbm_high_recall,
    year,
    serial,
```
